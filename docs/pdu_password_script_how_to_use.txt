How To Use bootstrap_pdu_passwords.py on Windows
===============================================

This guide explains how to install Python on Windows and run the Raritan PDU
admin password bootstrap script.

The script file is called:

    bootstrap_pdu_passwords.py

The default IP list file is called:

    pdus.txt

The script reads pdus.txt, connects to each PDU, and changes the admin password.

This guide is written for Windows PowerShell.

Before you start
----------------

You need:

    A Windows computer
    Network access to the PDU management IPs
    HTTPS access to the PDUs, usually TCP port 443
    The script file: bootstrap_pdu_passwords.py
    The current/default admin password for the PDUs
    The new admin password you want to set
    A text file named pdus.txt with one PDU IP address per line
    Python 3 installed
    The Python package named raritan installed

Recommended folder layout
-------------------------

Create one working folder and keep everything there.

Example:

    C:\pdu-bootstrap

Inside that folder, you should have:

    C:\pdu-bootstrap\bootstrap_pdu_passwords.py
    C:\pdu-bootstrap\pdus.txt

Step 1 - Install Python on Windows
----------------------------------

Use the official Python installer or Python install manager.

Recommended simple method:

1. Open this page in a browser:

       https://www.python.org/downloads/

2. Download Python for Windows.

3. Run the installer.

4. If the installer shows an option like this, enable it:

       Add python.exe to PATH

5. Finish the install.

Another supported method is the Python install manager from the Microsoft Store.
Python's current Windows documentation says the Python install manager can be
installed from the Microsoft Store app or downloaded from python.org/downloads.

Python.org also documents a WinGet install command for the Python install manager:

    winget install 9NQ7512CXL7T

Either method is fine as long as the python or py command works afterward.

Step 2 - Open PowerShell
------------------------

Open PowerShell.

A normal user PowerShell window is usually fine.

You do not usually need Administrator PowerShell unless your local computer has
restricted software install policies.

Step 3 - Check that Python works
--------------------------------

Run:

    py --version

If that works, you should see a Python version.

Example:

    Python 3.14.6

If py does not work, try:

    python --version

At least one of those should work.

If neither command works, Python is not installed correctly or is not on PATH.

Step 4 - Create the working folder
----------------------------------

Run:

    mkdir C:\pdu-bootstrap
    cd C:\pdu-bootstrap

Copy this script into that folder:

    bootstrap_pdu_passwords.py

Step 5 - Create pdus.txt
------------------------

Yes, you need to create the list of IPs.

The default filename is:

    pdus.txt

It must be in the same folder where you run the script, unless you pass a
different path with --ips.

Create the file:

    notepad .\pdus.txt

Put one PDU management IP address per line.

Example:

    10.42.98.10
    10.42.98.11
    10.42.98.12
    10.42.98.13
    10.42.98.14

Save the file and close Notepad.

Blank lines are OK.
Lines beginning with # are OK.
Duplicate IPs are skipped.

Example with comments:

    # Cabinet BJ16-BJ25 left-side PDUs
    10.42.98.10
    10.42.98.11
    10.42.98.12

    # Spare PDU
    10.42.98.13

Step 6 - Create a Python virtual environment
--------------------------------------------

This keeps the Raritan package installed only for this project folder.

Run:

    py -m venv .venv

Then activate it:

    .\.venv\Scripts\Activate.ps1

If PowerShell blocks activation, you may see an execution policy error.

If that happens, run:

    Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

Then try again:

    .\.venv\Scripts\Activate.ps1

When the virtual environment is active, your prompt usually starts with:

    (.venv)

Step 7 - Install the Raritan Python package
-------------------------------------------

With the virtual environment active, run:

    python -m pip install --upgrade pip
    python -m pip install raritan

Then test the import:

    python -c "import raritan.rpc; print('raritan import OK')"

If you see this, the package import worked:

    raritan import OK

The raritan package is the Xerus JSON-RPC Python binding for Raritan and Server
Technology devices.

Step 8 - First dry run
----------------------

Do a dry run before changing anything.

Run:

    python .\bootstrap_pdu_passwords.py --ips .\pdus.txt --dry-run -v

The script will ask for the current/default admin password.

For brand-new PX3-style Raritan PDUs, this may be:

    raritan

Type the password and press Enter.

The password will not show on screen while you type.

In dry-run mode, the script does not ask for the new password because it is not
changing anything.

Good dry-run output includes a timestamp, the log level, the PDU IP address,
and the result.

If pdus.txt has one PDU, good dry-run output looks like this:

    2026-07-03 21:56:58 INFO Processing 1 PDU(s) [DRY RUN, concurrency=1, verify_cert=False]...
    2026-07-03 21:56:58 INFO 10.42.98.137: OK - would_change
    2026-07-03 21:56:58 INFO Done. OK=1 Failed=0

If pdus.txt has more than one PDU, good dry-run output looks like this:

    2026-07-03 21:56:58 INFO Processing 5 PDU(s) [DRY RUN, concurrency=1, verify_cert=False]...
    2026-07-03 21:56:58 INFO 10.42.98.10: OK - would_change
    2026-07-03 21:56:59 INFO 10.42.98.11: OK - would_change
    2026-07-03 21:57:00 INFO 10.42.98.12: OK - would_change
    2026-07-03 21:57:01 INFO 10.42.98.13: OK - would_change
    2026-07-03 21:57:02 INFO 10.42.98.14: OK - would_change
    2026-07-03 21:57:02 INFO Done. OK=5 Failed=0

The exact date and time will be different on your computer.

The important part is this:

    <PDU IP address>: OK - would_change

That means the script can reach the PDU with the current/default admin password
and can proceed during a live run.

Step 9 - Live run
-----------------

After dry-run succeeds, run the live password change:

    python .\bootstrap_pdu_passwords.py --ips .\pdus.txt --log-file .\pdu-bootstrap.log

The script will ask for:

    Current/default admin password
    New admin password
    Confirm new admin password

The passwords will not show on screen while you type.

Good live output also includes a timestamp, the log level, the PDU IP address,
and the result.

If pdus.txt has one PDU, good live output looks like this:

    2026-07-03 22:05:12 INFO Processing 1 PDU(s) [LIVE, concurrency=1, verify_cert=False]...
    2026-07-03 22:05:14 INFO 10.42.98.137: OK - changed
    2026-07-03 22:05:14 INFO Done. OK=1 Failed=0

If pdus.txt has more than one PDU, good live output looks like this:

    2026-07-03 22:05:12 INFO Processing 5 PDU(s) [LIVE, concurrency=1, verify_cert=False]...
    2026-07-03 22:05:14 INFO 10.42.98.10: OK - changed
    2026-07-03 22:05:16 INFO 10.42.98.11: OK - changed
    2026-07-03 22:05:18 INFO 10.42.98.12: OK - changed
    2026-07-03 22:05:20 INFO 10.42.98.13: OK - changed
    2026-07-03 22:05:22 INFO 10.42.98.14: OK - changed
    2026-07-03 22:05:22 INFO Done. OK=5 Failed=0

The exact date and time will be different on your computer.

The important part is this:

    <PDU IP address>: OK - changed

That means the admin password was changed and the script verified that the new
password works.

Step 10 - Rerun if needed
-------------------------

The script is designed to be rerunnable.

If a PDU already has the new password, it should report a line like this:

    2026-07-03 22:08:33 INFO 10.42.98.137: OK - already_changed

That is not a problem.

It means the script tried the new password first and confirmed that the PDU was
already updated.

Common commands
---------------

Dry-run with the default pdus.txt:

    python .\bootstrap_pdu_passwords.py --dry-run -v

Live run with the default pdus.txt:

    python .\bootstrap_pdu_passwords.py --log-file .\pdu-bootstrap.log

Use a different IP list file:

    python .\bootstrap_pdu_passwords.py --ips .\my-pdus.txt --dry-run

Use a longer timeout:

    python .\bootstrap_pdu_passwords.py --ips .\pdus.txt --timeout 30 --dry-run

Process four PDUs in parallel:

    python .\bootstrap_pdu_passwords.py --ips .\pdus.txt --concurrency 4

For first use, keep concurrency at 1.

Use an environment variable for the old password
------------------------------------------------

This avoids typing the old password every run.

In PowerShell:

    $env:PDU_OLD_PASSWORD = "raritan"

Then run:

    python .\bootstrap_pdu_passwords.py --ips .\pdus.txt --dry-run

This environment variable only applies to the current PowerShell session.

Close PowerShell when you are done.

Avoid this when screen sharing or recording.

Command-line options
--------------------

--ips PATH

    IP list file.
    Default: pdus.txt

--old-password PASSWORD

    Old/current/default admin password.
    Supported, but not recommended because it can appear in shell history.

--timeout SECONDS

    Connection timeout per PDU.
    Default: 10

--concurrency NUMBER

    Number of PDUs to process in parallel.
    Default: 1

--dry-run

    Check connectivity and old password only.
    Does not change passwords.

--min-length NUMBER

    Local minimum length check for the new password.
    Default: 8

-v or --verbose

    Show debug logging.

--log-file PATH

    Write a log file.

--no-insecure

    Require valid trusted HTTPS certificates.

--insecure

    Disable HTTPS certificate verification.
    This is already the default because many PDUs use self-signed certificates.

What to do if something fails
-----------------------------

Problem:

    No IPs found in pdus.txt

Fix:

    Make sure pdus.txt exists in the folder where you are running the script.
    Make sure it has one IP address per line.

Problem:

    python is not recognized
    py is not recognized

Fix:

    Python is not installed correctly, or it is not on PATH.
    Reinstall Python and enable the PATH option if offered.

Problem:

    No module named raritan

Fix:

    The Raritan Python package is not installed in the Python environment you
    are using. Activate the virtual environment and run:

        python -m pip install raritan

Problem:

    HTTP Error 401 or HTTP Error 403

Fix:

    The old/current password is probably wrong for that PDU.
    Confirm the PDU's current admin password.

Problem:

    Connection timed out
    Connection refused
    No route to host

Fix:

    Check the IP address.
    Check that the PDU is powered and online.
    Check that your computer can reach the PDU management network.
    Check firewall rules for HTTPS/TCP 443.

Problem:

    password_needs_uppercase
    password_needs_number
    password_needs_special
    password_too_short

Fix:

    The new password does not meet the PDU password policy.
    Choose a stronger password and rerun the script.

Problem:

    Some PDUs changed and others failed

Fix:

    Review the failed IPs shown at the end.
    Fix the issue.
    Rerun the script.

    PDUs that already changed should report:

        OK - already_changed

Recommended first production process
------------------------------------

1. Put one test PDU in pdus.txt.

2. Run dry-run:

       python .\bootstrap_pdu_passwords.py --ips .\pdus.txt --dry-run -v

3. If dry-run works, run live:

       python .\bootstrap_pdu_passwords.py --ips .\pdus.txt --log-file .\pdu-bootstrap.log

4. Confirm you can log in to the PDU with the new password.

5. Add the remaining PDU IPs to pdus.txt.

6. Run dry-run again.

7. Run live again.

8. Save the log file for your records.

Security reminders
------------------

Do not paste real passwords into tickets, chat, email, or screenshots.

Do not use --old-password unless you have a specific reason.

Prefer the interactive password prompt.

Close PowerShell after using $env:PDU_OLD_PASSWORD.

Keep pdus.txt limited to the intended PDUs.

Review the log file before attaching or sharing it.
